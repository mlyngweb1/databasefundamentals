Objectives

In this lab, you can practice Data Defintion and Data Manipulation statements.

Database Schema

Team

  • teamId integer
  • teamname variable character, size 30
  • city variable character, size 20
  • ground variable character, size 25

Notes:

  • teamId is the primary key and is auto_increment.
  • teamname is unique and not null.
  • city and ground are not null also.

Player

  • playerId integer
  • fname variable character, size 20
  • lname variable character, size 20
  • position variable character, size 10
  • height tiny integer and is unsigned
  • weight small integer and is unsigned
  • salary decimal, size 8 including 2 decimal places and is unsigned
  • teamId integer

Notes:

  • playerID is the primary key and is auto_increment.
  • fname, lname, height, weight and salary are not null.
  • The height of a player is in inches while the weight is in pounds.
  • The teamId field is a foreign key to Team. If a teamId is updated in parent (Team) table, the teamId in the child table (Player) is also updated. If a teamId is deleted in parent (Team) table, the teamId in the child table (Player) is updated to null.
